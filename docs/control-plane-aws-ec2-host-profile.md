# AWS EC2 Control Plane Host Profile

The AWS EC2 profile generated by `deployment-control-plane setup --host-mode aws-ec2` is the
reviewed host realization path for one service and at least two workers.

## NixOS EC2

When viberoots controls the VM, use the generated NixOS EC2 wrapper. The wrapper imports the
existing `build-tools/tools/nix/deployment-control-plane-container-module.nix` module and supplies
AWS-specific image, instance, worker, artifact, and credential inputs. Do not copy that module into a
parallel EC2 implementation.

NixOS host evidence must include a reviewed NixOS AMI build/import identity and a pinned AMI
selection. Mutable marketplace aliases, `latest`-style selections, and unreviewed AMI owners are not
accepted.

## systemd/Podman Compatibility

The generated systemd unit files and `aws-ec2-podman-run.sh` are compatibility artifacts for
non-NixOS OCI hosts. They must preserve the same shared process and mount contract as the NixOS
module:

- one service process and at least two worker processes
- non-root runtime user where supported
- digest-pinned image reference
- read-only config and credential mounts
- persistent records, runtime, and artifact staging mounts
- service port binding only on the service process
- distinct worker ids
- restart policy, SIGTERM shutdown grace, and health/readiness command references

## Host Boundary

User data may only install or activate generated artifacts. AWS provider state changes must happen
through reviewed provider-capability hooks, not host bootstrap.

The launch template or Auto Scaling group must use selected private subnets, security groups, and
instance profile evidence from the reviewed AWS foundation profile. Registry runtime-pull proof must
come from the selected EC2 host path and match the exact image digest.

Operational visibility must include log routing, retention and access-control evidence, readiness
and worker-heartbeat history, and alarms for service down, readiness failure, missing worker
heartbeat, queue backlog, and repeated worker crash.
