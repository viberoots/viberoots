# helper-lib (Go library)

Generated by scaf (template: go/lib).

## Build and test

Run glue, then build and test:

```
build
verify //libs/helper-lib:helper-lib_test
```

## Add dependencies and refresh gomod2nix

Within this library directory:

```
cd libs/helper-lib
go get example.com/some/module@latest
go mod tidy
```

Then from repo root, regenerate glue and `gomod2nix.toml` deterministically:

```
tools/dev/install-deps.ts
```

This script will:

- Export the Buck graph, sync providers, and regenerate `third_party/providers/auto_map.bzl`.
- Generate or update `gomod2nix.toml` based on the current `go.mod`/`go.sum`.

## Patching a Go dependency

Use the repo patch tool to start a patch workspace, then apply a canonical patch:

```
tools/bin/patch-pkg start go golang.org/x/net
# Make edits in the workspace as directed by the tool
tools/bin/patch-pkg apply go golang.org/x/net
```

Patches are stored under `patches/go/` and wired via provider auto-map during glue.

## Repository layout

- `libs/helper-lib`: Library source, its `go.mod`, and `TARGETS` using `nix_go_library` and `nix_go_test`.
- `apps/*`: Example consumers of libraries; each owns its `TARGETS`.
- `patches/go/`: Canonical Go dependency patches created by `patch-pkg`.

## Providers and glue

`nix_go_*` macros attach providers automatically via the generated `third_party/providers/auto_map.bzl`.
Glue is generated by `tools/dev/install-deps.ts`, by CI, and also by repo wrappers like `build`.
