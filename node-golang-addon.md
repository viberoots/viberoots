## Node Golang Addon Scaffold — Design & Implementation Plan

This design adds a first‑class scaffolding template for a Node addon implemented in Go, built reproducibly via Buck2 + Nix, and aligned with our repository methodology and build‑system design.

- Audience: engineers and LLM agents implementing the scaffold and its tests
- Scope: Node TS package loading a `.node` addon; addon implemented in Go via `-buildmode=c-archive` with a tiny C N‑API binding; reproducible builds on aarch64‑darwin, aarch64‑linux, x86_64‑linux
- Non‑goals: exotic FFI stacks, vendoring, non‑N‑API shims, or per‑lockfile provider changes

### Why Go for a Node addon?

- Determinism: the Go toolchain is already pinned through Nix; cgo inputs can be provided via nixpkgs and repo libs.
- Performance: N‑API binding keeps calls in‑process with stable ABI guarantees; no runtime FFI package dependency.
- Simplicity: reuse existing planner capabilities (`goCArchive`, `cppNodeAddon`) and macros (`nix_go_carchive`, `nix_cpp_node_addon`) with precise invalidation and zero new provider shapes.

## Architecture (high‑level)

```mermaid
flowchart LR
  subgraph Workspace
    A["libs/<name> (Node TS)"]
    B["libs/<name>-go (Go, c-archive)"]
    C["libs/<name>-native (C N‑API shim)"]
  end

  A -->|requires| D["native/<addon_name>.node"]
  C -->|produces| D
  C -->|links| E["lib<go_pkg>.a (+ header)"]
  B -->|buildmode=c-archive| E

  subgraph Buck2 + Nix
    G["export-graph.ts → graph.json"]
    H["graph-generator.nix (planner)"]
    H -->|T.goCArchive| B
    H -->|T.cppNodeAddon (nixCxxPkgs includes goCArchive)| C
  end

  A -.->|nix_node_gen copies .node| D
```

Key points

- The Go package compiles as a C archive (`.a`) + header via `T.goCArchive`.
- The C binding compiles a Node‑API `.node` library via `T.cppNodeAddon`, linking the Go archive provided as a repo derivation.
- The Node TS package copies the `.node` artifact into a stable location (`native/<addon_name>.node`) for runtime loading.

---

## Alignment with project design

- Buck2 orchestrates graph/impact; Nix builds artifacts (dynamic derivations) — consistent with build‑system‑design.md.
- Package‑local patch directories (Go) are included in `srcs` so patch edits precisely invalidate consumers.
- Node remains “macro‑only”; importer‑scoped provider flow is unchanged (no new provider shapes).
- Dev overrides (`NIX_GO_DEV_OVERRIDE_JSON`) are allowed locally and forbidden in CI; planner templates emit warnings locally and fail in CI.

## Scaffold contents and layout

Template root

- `tools/scaffolding/templates/node/go-addon/`
  - Generates three sibling packages:
    - `libs/<name>`: Node TS library that loads the addon
    - `libs/<name>-go`: Go package compiled as a C archive (c-archive)
    - `libs/<name>-native`: C N‑API binding that links the Go archive and produces the `.node`

Copier variables (illustrative; match existing Node scaffolds)

- `name` (required): base name for all three packages
- `addon_name` (optional): defaults to `<name>_addon`
- `pkgScope` (optional): npm scope for the Node package
- `includeNodeTests` (optional, default true)
- `importer` and `lockfilePath` (derived/conventional): used to stamp `lockfile:<path>#<importer>` labels in Node macros

Generated files (high‑level)

- `libs/<name>/` (Node package)
  - `package.json.jinja`, `tsconfig.json.jinja`, `src/index.ts.jinja`, `test/index.test.ts.jinja` (optional), `TARGETS.jinja`, `README.md.jinja`
- `libs/<name>-go/` (Go package, c-archive)
  - `go.mod.jinja`, `gomod2nix.toml.jinja` (placeholder, regenerated by install‑deps), `pkg/addon/addon.go.jinja` (exports a C API), `TARGETS.jinja`, `patches/go/.keep`
- `libs/<name>-native/` (C N‑API binding)
  - `include/<name>_binding.h.jinja`, `src/binding.c.jinja` (N‑API shim calling Go C API), `TARGETS.jinja`, `tests/<name>_gtest.cpp.jinja` (optional), `patches/cpp/.keep`

## Buck targets and macros (scaffolded)

Go c-archive (planner‑visible; no direct link in Buck)

```
load("//go:defs.bzl", "nix_go_carchive")

nix_go_carchive(
    name = "carchive",
    srcs = glob(["pkg/**/*.go"]),
    labels = ["lang:go", "kind:carchive"],
    # Include local patches for precise invalidation
    # (macro appends patches/go by default via append_patch_srcs under the hood if used elsewhere)
    visibility = ["PUBLIC"],
)
```

C N‑API binding → .node (links the Go c-archive via planner)

```
load("//cpp:defs.bzl", "nix_cpp_node_addon", "nix_cpp_test")

nix_cpp_node_addon(
    name = "napi_addon",
    srcs = [
        "src/binding.c",
    ],
    headers = glob(["include/**/*.h"]),
    deps = ["//libs/<name>-go:carchive"],  # planner will link T.goCArchive output
    addon_name = "<addon_name>",
    labels = ["lang:cpp", "kind:addon"],
    visibility = ["PUBLIC"],
)
```

Node package (stable path copy + library/test)

```
load("//node:defs.bzl", "nix_node_lib", "nix_node_test", "nix_node_gen")

nix_node_gen(
    name = "copy_addon",
    out = "native/<addon_name>.node",
    cmd = "cp $(location //libs/<name>-native:napi_addon) $OUT",
    lockfile_label = "lockfile:<lockfilePath>#<importer>",
    labels = ["lang:node", "kind:gen"],
)

nix_node_lib(
    name = "<name>",
    srcs = [],
    out = "build.stamp",
    cmd = "echo ok > $OUT",
    deps = [":copy_addon"],
    lockfile_label = "lockfile:<lockfilePath>#<importer>",
)

# Exactly one test per file (per repo convention)
nix_node_test(
    name = "unit",
    deps = ["//libs/<name>-native:napi_addon"],
    lockfile_label = "lockfile:<lockfilePath>#<importer>",
)
```

## Minimal code skeletons (scaffold)

Go (export C API; compiled as c-archive)

```go
// pkg/addon/addon.go
package addon

/*
#include <stdint.h>
*/
import "C"

//export Add
func Add(a, b C.int) C.int {
  return a + b
}

// Note: buildmode=c-archive emits lib<module>.a and a header with exported symbols.
```

C N‑API binding (calls into the Go C API)

```c
// src/binding.c
#include <node_api.h>
#include "<go_module>.h"  // generated by c-archive; scaffold wires include path via planner

static napi_value AddWrapped(napi_env env, napi_callback_info info) {
  size_t argc = 2;
  napi_value argv[2];
  napi_get_cb_info(env, info, &argc, argv, NULL, NULL);
  int32_t a = 0, b = 0;
  napi_get_value_int32(env, argv[0], &a);
  napi_get_value_int32(env, argv[1], &b);
  int32_t sum = Add(a, b);
  napi_value out;
  napi_create_int32(env, sum, &out);
  return out;
}

static napi_value Init(napi_env env, napi_value exports) {
  napi_value fn;
  napi_create_function(env, "add", NAPI_AUTO_LENGTH, AddWrapped, NULL, &fn);
  napi_set_named_property(env, exports, "add", fn);
  return exports;
}

NAPI_MODULE(NODE_GYP_MODULE_NAME, Init)
```

Node loader (stable relative path)

```ts
// src/index.ts
import { createRequire } from "module";
import path from "node:path";
import { fileURLToPath } from "node:url";

const requireFromHere = createRequire(import.meta.url);
const __dirname = path.dirname(fileURLToPath(import.meta.url));

const override = process.env.ADDON_PATH || "";
const addonPath = override || path.join(__dirname, "native/<addon_name>.node");

const addon = requireFromHere(addonPath);
export const add = (a: number, b: number) => addon.add(a, b);
```

## Planner behavior (already present; no new shapes)

- Go plugin
  - `nix_go_carchive` stamps `lang:go` + `kind:carchive` on the Buck node.
  - `T.goCArchive` builds the C archive derivation and exports `lib/` and `include/` outputs.
- C++ plugin
  - For `kind:addon` targets, scans direct `deps` for `kind:carchive` nodes.
  - Calls `T.goCArchive` on those deps and passes the resulting derivations as `nixCxxPkgs` into `T.cppNodeAddon`.
  - The addon template (`cppNodeAddon`) auto‑discovers static libraries in `…/lib` to link and includes `…/include` during compilation.
- Node macros
  - No changes; importer‑scoped provider mapping is unchanged. The `.node` artifact is copied by `nix_node_gen` to a deterministic path.

Result: No provider/auto_map changes; the scaffold composes existing planner templates.

---

## Patching and dev overrides

- Go patches: package‑local `libs/<name>-go/patches/go/*.patch`; included in `srcs` for precise invalidation.
- Node patches: importer‑local (unchanged); the Node wrapper TARGETS includes the lockfile label.
- Dev overrides: `NIX_GO_DEV_OVERRIDE_JSON` supported by the Go templates; warn locally, forbidden in CI.

## Tests and acceptance

Unit tests (scaffolded)

- Node: `//libs/<name>:unit` — calls `add(2, 3)` and expects `5` (exactly one test per file).
- (Optional) C++ gtest in `libs/<name>-native` for minimal compile sanity; Node test is sufficient as the user‑facing proof.

Acceptance criteria

- `scaf new node go-addon <name>` creates `libs/<name>`, `libs/<name>-go`, `libs/<name>-native`.
- `buck2 build //libs/<name>-go:carchive` succeeds.
- `buck2 build //libs/<name>-native:napi_addon` succeeds (links the Go archive).
- `node tools/buck/export-graph.ts && node tools/buck/gen-auto-map.ts …` complete without provider diffs.
- `buck2 test //libs/<name>:unit` passes.

## CI integration

Stages (unchanged from repo design)

1. Export Graph — `tools/buck/graph.json`
2. Sync Providers — Node only (no new shapes); noop if no lockfiles
3. Generate auto_map — unchanged
4. Build & Test — build `//libs/<name>-native:napi_addon` and run Node test
5. Pre-build guard — unchanged (glue presence/freshness)

Matrix: aarch64‑darwin, aarch64‑linux, x86_64‑linux (ensures Go + N‑API flags are portable)

## Troubleshooting

- “Symbol not found” at runtime: ensure the C binding includes the generated Go header and the addon target depends on `//libs/<name>-go:carchive`.
- Darwin linkage quirks: the addon template links with `-dynamiclib -undefined dynamic_lookup`; avoid relying on transitive resolution of non‑Node libs; use explicit nixpkgs deps when needed.
- Dev overrides are set: `tools/dev/clear-overrides.ts` unsets `NIX_GO_DEV_OVERRIDE_JSON` for clean, cacheable builds.
